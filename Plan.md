# План разработки программы на Ruby для публикации постов в WordPress и VKontakte

## 1. Обзор задачи
Создать программу на Ruby, которая:
- Предоставляет циклическое меню выбора платформы для публикации (WordPress, VKontakte)
- После каждой публикации возвращает пользователя в меню для новых выборов
- При выборе завершения выводит отчет обо всех проведенных публикациях
- Принимает путь до директории с публикуемыми файлами как второй параметр командной строки
- Сканирует указанную директорию на наличие файлов
- Находит единственное изображение-обложку (с "_logo." в имени)
- Находит единственный текстовый файл *.txt
- Обрабатывает вставки "[имя файла]" в тексте для размещения медиа в указанных местах
- Загружает изображения и видео в тело поста согласно вставкам в тексте
- Файлы, не указанные в тексте, вставляются после основного текста перед футером
- Читает текст новости из *.txt файла (первая строка - заголовок, остальные - абзацы)
- Публикует пост на WordPress через gem `rubypress` (код вынесен в отдельный модуль)
- Публикует пост в VKontakte через API (код вынесен в отдельный модуль)
- Хранит учетные данные в отдельном файле конфигурации
- Поддерживает расширение для публикации на другие платформы в будущем

## 2. Структура проекта
```
/
├── Plan.md                    # Этот файл
├── config.yml                 # Файл конфигурации (хост, логин, пароль для WordPress и VK)
├── herausgeber.rb             # Основная программа (точка входа)
├── wordpress_publisher.rb     # Модуль для работы с WordPress (rubypress)
├── vk_publisher.rb            # Модуль для работы с VKontakte (VK API)
└── [папка с контентом]        # Папка с файлами для публикации (передается параметром)
    ├── *_logo.*               # Изображение-обложка (единственный экземпляр)
    ├── *.jpg, *.png           # Другие изображения
    ├── *.mp4, *.mov           # Видеофайлы
    └── *.txt                  # Текст новости (единственный экземпляр)
```

## 3. Этапы реализации

### Этап 1: Подготовка окружения
- Установить Ruby (если не установлен)
- Инициализировать Gemfile
- Добавить зависимость `rubypress`
- Добавить зависимость `yaml` для работы с конфигом

### Этап 2: Создание файла конфигурации
- Создать `config.yml` с полями:
  - `host`: URL WordPress сайта (XML-RPC endpoint)
  - `username`: логин пользователя
  - `password`: пароль пользователя
- Добавить `.gitignore` для исключения config.yml из репозитория

### Этап 3: Реализация основных модулей

#### 3.1 Модуль конфигурации
- Чтение и парсинг config.yml
- Валидация наличия всех необходимых полей
- Обработка ошибок при отсутствии файла

#### 3.2 Модуль работы с файлами (в herausgeber.rb)
- Прием пути к директории из командной строки (второй параметр)
- Сканирование указанной директории
- Поиск единственного файла-обложки (по паттерну "*_logo.*")
  - Ошибка при отсутствии или множественном нахождении
- Поиск единственного текстового файла (*.txt)
  - Ошибка при отсутствии или множественном нахождении
- Сбор остальных изображений и видеофайлов
- Фильтрация по расширениям:
  - Изображения: .jpg, .jpeg, .png, .gif, .webp
  - Видео: .mp4, .mov, .avi, .wmv

#### 3.3 Модуль обработки текста (в herausgeber.rb)
- Чтение *.txt файла
- Первая строка → заголовок поста
- Остальные строки → абзацы тела поста
- Обработка пустых строк
- Поиск вставок формата "[имя файла]" в тексте
- Замена вставок на соответствующие блоки изображений/видео
- Файлы, не упомянутые в тексте, добавляются после всего текста перед футером

#### 3.4 Модуль меню (в herausgeber.rb)
- Вывод интерактивного циклического меню выбора платформы
- Пункты меню:
  - "1. Опубликовать новость на сайт pnisurov.ru"
  - "2. Опубликовать новость в VKontakte (vk.com)"
  - "0. Завершить работу и показать отчет"
- После каждой публикации возврат в меню для нового выбора
- При выборе завершения - вывод отчета о всех проведенных публикациях

#### 3.5 Модуль работы с WordPress (wordpress_publisher.rb)
- Вынесен в отдельный файл для поддержки других платформ
- Инициализация соединения с сайтом через rubypress
- Загрузка медиафайлов:
  - Обложка → featured image
  - Остальные изображения/видео → вставка в тело поста
- Формирование контента поста со строгой HTML-разметкой для WordPress Gutenberg:
  - **Абзацы**: использование специального блока `<!-- wp:paragraph -->` с классами `ajustify indent` и отступами
  - **Изображения**: вставка через блок `<!-- wp:group -->` с вложенным `<!-- wp:image -->`, включая ID изображения, URL, стили границ (рамка 2px #6bcae1, радиус 8px) и отступы
  - **Вставки "[имя файла]"**: поиск в тексте, проверка наличия файла, замена на соответствующий блок изображения или видео
  - **Видео**: вставка через блок `<!-- wp:group -->` с вложенным `<!-- wp:video -->`, включая стили границ (рамка 3px #a54200, радиус 5px), тень и атрибуты видео
  - **Обработка множественных видео**: первое видео имеет атрибут `autoplay`, остальные — только `controls`
  - **Футер новости**: добавление в конец поста разделителя (`<!-- wp:separator -->`) и абзаца со ссылкой на канал в мессенджере MAX
  - **Файлы без вставок**: файлы, не указанные в тексте через "[имя файла]", вставляются после всего текста перед футером
- Возврат результата в основной модуль (URL опубликованного поста)

#### 3.6 Модуль работы с VKontakte (vk_publisher.rb)
- Вынесен в отдельный файл для модульности
- Инициализация соединения с VK API через токен доступа
- Загрузка медиафайлов (изображения и видео) через VK API методы:
  - `photos.getWallUploadServer` + `photos.saveWallPhoto` для изображений
  - `video.save` для видео
- Требования к публикации:
  - Текст загружается без форматирования, между абзацами отступ в строку
  - Первая строка текста - заголовок новости
  - В конце поста добавляется: "------------------------------------------------\nБольше новостей — в нашем канале в мессенджере MAX: https://max.ru/id3430030612_gos"
  - Параметр `primary_attachments_mode` устанавливается в значение `grid`
  - Параметр `from_group` устанавливается в значение `1`
  - Количество прикрепляемых изображений и видео - не более 10
  - Изображение с `_logo` загружается первым среди всех медиафайлов
  - Если у новости кроме заголовка нет текста, а есть только видео - изображение с `_logo` не загружать, грузить только видео
- Возврат результата в основной модуль (URL опубликованного поста)

### Этап 4: Обработка ошибок и логирование
- Обработка ошибок подключения к WordPress

### Этап 5: Тестирование

  - Отсутствует обложка
  - Отсутствует txt файл
  - Пустая папка

  - Публикация в VKontakte
  - Циклическое меню (несколько публикаций подряд)
  - Завершение работы и вывод отчета

## 4. Технические детали

### Зависимости (Gemfile)
```ruby
source 'https://rubygems.org'

gem 'rubypress'
gem 'yaml'
gem 'httparty'  # Для работы с VK API
```

### Структура config.yml
```yaml
wordpress:
  host: "https://yoursite.com/xmlrpc.php"
  username: "your_username"
  password: "your_password"

vkontakte:
  access_token: "your_access_token"
  group_id: "your_group_id"  # ID группы для публикации
```

### Алгоритм работы программы
1. Загрузить конфигурацию из config.yml
2. Проверить наличие параметра пути к директории в командной строке
3. Если путь не указан - вывести справку и завершить работу
4. Перейти в указанную директорию
5. Найти все файлы в директории
6. Идентифицировать:
   - Обложку (единственный файл с "_logo." в имени)
   - Текстовый файл (единственный *.txt)
   - Медиафайлы (остальные изображения и видео)
7. При отсутствии или множественном нахождении обложки/txt - вывести ошибку
8. Прочитать и распарсить текстовый файл
9. Показать циклическое меню выбора платформы публикации:
   - "1. Опубликовать новость на сайт pnisurov.ru"
   - "2. Опубликовать новость в VKontakte (vk.com)"
   - "0. Завершить работу и показать отчет"
10. При выборе "Опубликовать новость на сайт pnisurov.ru":
    - Инициализировать соединение с WordPress через rubypress (из wordpress_publisher.rb)
    - Загрузить обложку и получить ID
    - Загрузить остальные медиафайлы и получить URLs/ID
    - Сформировать HTML-контент поста согласно шаблонам Gutenberg:
      - Поиск вставок "[имя файла]" в тексте и замена на соответствующие блоки медиа
      - Файлы, не указанные в тексте, вставляются после основного текста перед футером
      - Заголовок из первой строки txt
      - Абзацы текста с блоками `<!-- wp:paragraph -->`
      - Изображения с блоками `<!-- wp:group -->` и `<!-- wp:image -->`
      - Видео с блоками `<!-- wp:group -->` и `<!-- wp:video -->` (autoplay только для первого)
      - Добавить футер с разделителем и ссылкой на MAX
    - Создать пост с установленной обложкой
    - Сохранить информацию о публикации (название новости, URL) для отчета
    - Вернуться в меню
11. При выборе "Опубликовать новость в VKontakte":
    - Инициализировать соединение с VK API (из vk_publisher.rb)
    - Подготовить текст новости (заголовок + абзацы с отступами)
    - Загрузить медиафайлы (не более 10):
      - Если есть только видео и нет текста кроме заголовка - загружать только видео

    - Опубликовать пост с параметрами primary_attachments_mode=grid, from_group=1
    - Добавить футер с ссылкой на MAX
    - Сохранить информацию о публикации (название новости, URL) для отчета
    - Вернуться в меню
12. При выборе "Завершить работу":
    - Вывести отчет о всех проведенных публикациях в формате:

## 5. Потенциальные проблемы и решения

| Проблема | Решение |
|----------|---------|
| Несколько файлов *_logo.* | Прервать выполнение с ошибкой (требуется единственный экземпляр) |
| Несколько *.txt файлов | Прервать выполнение с ошибкой (требуется единственный экземпляр) |
| Отсутствие обложки | Прервать выполнение с ошибкой |
| Отсутствие txt файла | Прервать выполнение с ошибкой |
| Не указан путь к директории | Вывести справку по использованию |
| Вставка "[имя файла]" не соответствует файлу | Пропустить вставку, продолжить обработку |
| Файл указан в тексте несколько раз | Вставить в каждом указанном месте |
| Ошибка загрузки медиа | Продолжить с остальными файлами, залогировать ошибку |
| Большие файлы видео | Предупредить о лимитах WordPress |
| Ошибка подключения к WordPress | Вывести понятное сообщение, предложить проверить конфиг |
| Ошибка подключения к VK API | Вывести понятное сообщение, предложить проверить токен доступа |
| Превышение лимита 10 медиафайлов для VK | Обрезать список до 10 файлов, logo всегда первый |

## 8. Детали формирования контента (HTML-шаблоны Gutenberg)

### 8.1 Абзацы текста
Каждый абзац из txt-файла оборачивается в блок:
```html
<!-- wp:paragraph {"className":"ajustify indent","style":{"spacing":{"margin":{"top":"var:preset|spacing|30","bottom":"var:preset|spacing|30"}}}} -->
<p class="ajustify indent" style="margin-top:var(--wp--preset--spacing--30);margin-bottom:var(--wp--preset--spacing--30)">текст_абзаца</p>
<!-- /wp:paragraph -->
```

### 8.2 Изображения
Каждое изображение (кроме обложки) вставляется как группа с изображением:
```html
<!-- wp:group {"metadata":{"categories":[],"patternName":"core/block/3427","name":"Изображение с рамками"},"style":{"spacing":{"margin":{"top":"var:preset|spacing|30","bottom":"var:preset|spacing|30"}}},"layout":{"type":"constrained"}} -->
<div class="wp-block-group" style="margin-top:var(--wp--preset--spacing--30);margin-bottom:var(--wp--preset--spacing--30)"><!-- wp:image {"id":ID_изображения,"sizeSlug":"large","linkDestination":"none","align":"wide","className":"has-lightbox","style":{"border":{"radius":"8px","color":"#6bcae1","width":"2px"},"spacing":{"margin":{"top":"var:preset|spacing|40","bottom":"var:preset|spacing|40"}}}} -->
<figure class="wp-block-image alignwide size-large has-custom-border has-lightbox" style="margin-top:var(--wp--preset--spacing--40);margin-bottom:var(--wp--preset--spacing--40)"><img src="URL изображения" alt="" class="has-border-color wp-image-5178" style="border-color:#6bcae1;border-width:2px;border-radius:8px"/></figure>
<!-- /wp:image --></div>
<!-- /wp:group -->
```
Где `ID_изображения` и `URL изображения` подставляются после загрузки файла на WordPress.

### 8.3 Видео
Каждое видео вставляется как группа с видео:
```html
<!-- wp:group {"metadata":{"categories":[],"patternName":"core/block/4274","name":"Видео"},"style":{"border":{"radius":"5px","color":"#a54200","width":"3px"},"spacing":{"padding":{"top":"0","bottom":"0","left":"0","right":"0"},"margin":{"top":"var:preset|spacing|40","bottom":"var:preset|spacing|40"},"blockGap":"0"},"dimensions":{"minHeight":"0px"},"shadow":"var:preset|shadow|deep"},"layout":{"type":"constrained"}} -->
<div class="wp-block-group has-border-color" style="border-color:#a54200;border-width:3px;border-radius:5px;min-height:0px;margin-top:var(--wp--preset--spacing--40);margin-bottom:var(--wp--preset--spacing--40);padding-top:0;padding-right:0;padding-bottom:0;padding-left:0;box-shadow:var(--wp--preset--shadow--deep)"><!-- wp:video {"id":ID видео,"style":{"spacing":{"margin":{"top":"0","bottom":"0","left":"0","right":"0"}}}} -->
<figure style="margin-top:0;margin-right:0;margin-bottom:0;margin-left:0" class="wp-block-video"><video autoplay controls src="URL видео" playsinline></video></figure>
<!-- /wp:video --></div>
<!-- /wp:group -->
```
**Важно**: 
- Первое видео содержит атрибуты `autoplay controls`
- Последующие видео содержат только `controls` (без `autoplay`)

### 8.4 Футер новости
В конце каждого поста добавляется:
```html
<!-- wp:separator {"style":{"spacing":{"margin":{"top":"var:preset|spacing|40","bottom":"var:preset|spacing|40"}}}} -->
<hr class="wp-block-separator has-alpha-channel-opacity" style="margin-top:var(--wp--preset--spacing--40);margin-bottom:var(--wp--preset--spacing--40)"/>
<!-- /wp:separator -->

<!-- wp:paragraph -->
<p><em>Более подробная информация и актуальные новости доступны в нашем официальном канале в мессенджере <strong><a href="https://max.ru/id3430030612_gos ">MAX</a></strong></em></p>
<!-- /wp:paragraph -->
```
